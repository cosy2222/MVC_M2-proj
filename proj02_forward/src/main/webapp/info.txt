

서블릿 포워드 (forward) 기능 ( 다양한 방법의 forward 기능 사용하기 ) 
	포워드 : 하나의 서블릿에서 다른 서블릿이나 jsp로 연동하는 방법
	 	( 페이지 이동 , 변수의 값을 다른페이지로 이동) 
	 	
	포워드의 사용 용도
		1. 요청에 대한 추가 작업을 다른 서블릿에게 수행하게 합니다
		2. 요청에 포함한 정보를 다른 서블릿이나 jsp와 공유할 수 있다
		3. 요청 정보를 포함 시켜 다른 서블릿에게 전달 할 수 있다
		4. MVC M2 개발시 서블릿에게 jsp페이지로 데이터를 전달 할떄 사용 
		
	서블릿 (Controller) 에서 포워드 방법에는 4가지
	
	1. redirect 방법
	 	Response 객체의 sendRedirect() 메소드를 이용
	 	웹브라우저에서 재요청하는 방법
	 	Response.sendRedirect("포워드 서블릿 또는 JSP");
	 	URL주소가 바뀜
	
	2. refresh 방법
		Response 객체의 addHeader() 메소드를 이용
		웹브라우저에서 재요청하는 방법
		Response.addHeader ("Refresh , 경과시간(초); url=요청할 서블릿이나 JSP");
		URL주소가 바뀜
	
	3. location 방법
		자바스크립트의 location 객체의 href속성을 이용하는 방법
		자바스크립트에서 재요청하는 방법
		location.href = ' 요청 서블릿 또는 JSP ' ;
		URL주소가 바뀜
	
		1,2,3 방법 : client 가 서버로 페이지를 새롭게 요청 하는 방법
				- Get 방식으로 변수의 값을 다른 페이지로 전달 ( 전송 용량의 한계를 가지고 있다 ) 
		
	4. dispatch 방법 (일반적인 포워딩 : 서버에서 이동) : URL 주소가 바뀌지 않는다
			<== 서버에서 클라이언트의 요청 작업을 마치고 view페이지로 이동할때 많이사용
			request.setAttribute () : 변수의 값 할당
			request.getAttribute () : 변수의 값 가져오기 
			대량의 변수의 값을 처리가능 
	
		서블릿이 직접 요청하는 방법
		RequestDispatcher 클래스의 forward() 메소드를 이용함
		RequestDispatcher dis = request.getRequestDispacher("포워드할 서블릿이나 JSP");
		dis.forward(request , response); 
	
        4방법: 서버에서 서버의 페이지를 요청
